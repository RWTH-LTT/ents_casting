## What is ents_casting?

`ents_casting` generates long-term scenarios for different weather years based on measured data of a single year and historical weather data. The long-term scenarios can be used to analyze the energy system operation under various weather conditions.

For each day in the long-term scenarios, `ents_casting` generates a stochastic forecast of time series parameters with a forecast horizon of 6 days. The stochastic forecasts are based on synthetic weather forecasts, which mimic the accuracy of historical weather forecasts. With the stochastic forecasts, the energy system operation can be simulated and different operation methods can be benchmarked, e.g., with [s2_mpc](https://github.com/RWTH-LTT/s2_mpc).

Currently supported time series parameters:

- `pv_local`: Capacity factor for photovoltaics.
- `wind_local`: Capacity factor for wind turbines.
- `demand_el`: Electricity demand.
- `demand_heat`: Heating demand.
- `demand_cold`: Cooling demand.
- `price_el`: Electricity price.
- `emissions_el`: Electricity emission factor.

Inputs:

- Measured time series data of each parameter for one year.

Outputs:

- Long-term scenarios: Each scenario contains the time series of all parameters over one year. A scenario is generated by learning the relation between relevant weather parameters and the measured time series, then predicting the time series for other weather years. ents_casting then adds a synthetic error term to account for imperfect predictive capabilities.
- Stochastic forecasts: Forecasts of all parameters are made at the start of each day in the long-term scenarios. The forecast horizon is 6 days with hourly resolution.

## Installation

Clone the git repository:

```bash
git clone https://github.com/RWTH-LTT/ents_casting.git
```

Create the venv:

```bash
python3 -m venv ./venv_s2_mpc
```

Activate the venv:

```bash
. venv_s2_mpc/Scripts/activate
```

Install `ents_casting` and all required packages:

```bash
pip install -e ./ents_casting
```

## Configure `config.yaml`

The file `config.yaml` contains all information about which data is used for scenario and forecast generation. The parameters are documented in the file.

## Provide historical measured data for your case study

You can provide your own historical measured data by copying them into `data/measured_data/data_{year}.csv`. They year depends on the `training_years` specified in `config.yaml`. You can use different years of measured data for different parameters.

## Run the main entry point (or individual steps within it)

```bash
python -m ents_casting
```

The main script runs, in order:

1. Download the historical weather data for the specified weather years and locations.
2. Download the historical weather forecasts.
3. Train the LightGBM forecast models, including k-fold cross-validation.
4. Generate the long-term scenarios for the specified weather years using the trained LightGBM models.
5. Fit AR models to the error between the generated long-term scenarios and the measured data.
6. Add AR noise to the long-term scenarios and plot the long-term scenarios.
7. Train the AR error models for weather forecasts.
8. Train the AR error models for residuals between the LightGBM forecasts and measured data.
9. Generate the short-term forecasts for each day of the long-term scenarios and plot the stochastic forecasts.

When using many (>10) historical weather years, API limitations may lead to errors when downloading the weather data. In that case, reduce the number of locations for which the weather data should be downloaded. In this way, you can download the data in multiple steps.
